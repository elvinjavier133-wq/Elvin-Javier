# 🛒 Temu – Functional Purchase Flow Testing  
**Proyecto educativo para portafolio QA – Elvin Javier Rosa**

Este proyecto forma parte de mi portafolio como **QA Tester Junior** y fue realizado con fines totalmente **educativos**, con el objetivo de practicar el diseño de casos de prueba, validación funcional y documentación del flujo de compra en una aplicación móvil.

---

##  Objetivo del Proyecto  
Validar el **flujo completo de compra** dentro de la aplicación Temu, asegurando que el sistema permita realizar correctamente cada paso e identifique errores cuando se ingresan datos inválidos o incompletos.

---

##  Tipos de Pruebas Realizadas
- Pruebas funcionales  
- Pruebas negativas  
- Validación de UI  
- Validación de mensajes de error  
- Validación de comportamiento del sistema  

---

## 📘 Alcance del Testing  
Se evaluaron las siguientes funcionalidades:

- Búsqueda de productos  
- Visualización del detalle del producto  
- Agregar productos al carrito  
- Ver carrito de compras  
- Proceso de checkout  
- Autenticación requerida  
- Validación de método de pago  
- Escenarios negativos (datos inválidos)  

---
